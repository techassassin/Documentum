To use the configuration matrix:


Create a configuration component:
Configuration components are the types of configurations, such as autonaming and security. Specific instances of configurations are saved as configuration elements, shown as collapsible rows in the vertical Y-axis of the configuration matrix. For example, you can create a security element for quality assurance draft writers with WRITE access. You can access the element by expanding the Security configuration component.


Create a context:
Shown as the horizontal X-axis of the configuration matrix. Contexts comprise of logical groupings of content qualified by conditions and composed of properties such as document types, DQL conditions, and user groups. Contexts determine when a configuration element applies to content within the repository. For example, you can create a context that applies for all dm_document content types created for quality documents and set to the Draft status.


Apply the configuration element to the context
For example, applying the configuration element to the context created in the first two examples leads to a configuration such that the Draft dm_document quality assurance content provides quality assurance draft writers WRITE access.


The D2 engine determines which config, that is autolinking, check-in, security, and other such component configurations to use, by scanning all contexts defined in the matrix from left to right. The first context which is matched will result in the D2 engine applying all component configurations associated to the same context from top to bottom.

    A D2 Dictionary is a collection of key values and one or more sets of descriptive labels, used to provide value assistance to the end user.

    Dictionaries help end users by defining choice sets for property values, or say metadata for generating autonaming patterns, and providing input assistance during advanced searches.

    The advantages of dictionaries that it increases the likelihood of a property being filled out by an end user.

    Reduce labeling errors such as extraneous spaces, inconsistent plurality, and so on.

    You can use an alias to create an alternate equivalent of the value. For example, you can have the value True and False, and assign as aliases the values Yes and No, 0 and 1 and In Progress. In configuration components such as property page fields, you can designate the dictionary and alias used. You can use the different terms to assist end users. For example, in some cases when an end user accesses a list box, they see the option as True/False. You can use aliases so that in other cases, the end user sees the option Yes and No. Despite the difference, both terms are treated as equivalent values and receive equivalent treatment from automatic processes.

    Understanding Minimum Security Rights

    By default, the admingroup group or the Documentum System administratorÃ¢â¬â¢s group is created during D2 installation or upgrade and contains all Superusers in the repository.
    When you grant Superuser privileges to a user, you may also need to add that user to the admingroup group to enable them to run jobs or administration methods. Users belonging to the admingroup group have no inherent privileges other than those they have as Superusers. However, administrative jobs, methods, and other related objects use an ACL which restricts their use to the admingroup group.
    If you revoke Superuser privileges from a user, the user should also be removed from the admingroup group.

    Note: Do not delete the admingroup group from the repository. The admingroup group, which includes all Superusers within the repository, is used by the jobs that make up the administration tool suite.

    Editing a Security Model

    This is the trickiest thing that you do on the D2 environment. Updating the Security model affects the newly created document as well as the previously created documents

    1. To modify a security model and affect existing content, modify the security model and do not modify the configuration matrix.
      D2 runs a job in the background to modify the ACL of all previously created content using the scurity model.
    2. To modify a security model and avoid affecting existing content, unless you perform a manual operation:
      • Create a copy of the security model.
      • In the configuration matrix, unselect the previous security model and select the new security model.

      D2 applies the security model to new content and to existing content when it is saved, checked in,or has a property changed.
    3. To modify a security model and avoid affecting existing content:
      1. Create a copy of the security model.
      2. Create a copy of the content context used for the previous security model. Add the condition r_creation_date > <the date of creation for the new security model>to identify the content created for the new security model.
      3. Drag the new context before the old context in the configuration matrix.
      4. Activate the new context.

        5. D2 applies the new model to all new content, and old content continues using the old security model. You can launch D2 Core Job using Documentum Administrator or run a mass update to apply the new security model to linked content through a D2 Core Job DQL

    Uniqueness Check

    Uniqueness Checks has been created to safeguard and restrict user from sending incomplete data to LMS system.

    These Uniqueness Checks come into action only when the Training Required field at time of Document Creation is selected as True

    Click on each of the tabs below to find more details on these Uniquness Checks

    • SEQ LMS Curriculum fields are not filled
    • SEQ LMS Quiz Name has associated Relation
    • SEQ LMS Validate Quiz has only one Relation

    The "SEQ LMS Curriculum fields are not filled" Uniqueness check is created to restrict user from initiating a workflow if the Curriculum ID field is empty

    Below DQL Predicate is used in this Uniqueness Check

    dm_docbase_config where not exists (select seq_lms_curriculum_id FROM cd_quality_gmp WHERE (r_object_id = '$value(r_object_id)' and any seq_lms_curriculum_id is not nullstring and training_required = 1) or (r_object_id = '$value(r_object_id)' and training_required = 0))

    If the Curriculum ID is not provided by the user, while initiating a workflow, following message will be displayed

    Curriculum Name in the Training Info tab is not updated. Update all the fields before sending the document(s) into Workflow.

    TheSEQ LMS Quiz Name has associated Relation Uniqueness check is created to restrict user from initiating a workflow, if he select a different Quiz Name in the Training Info tab and creating a relation with a different Quiz Document.

    Below DQL Predicate is used in this Uniqueness Check

    dm_docbase_config where not exists (select r_object_id from cd_quality_gmp WHERE (seq_lms_material_type!='SOP' and r_object_id = '$value(r_object_id)' and training_required = 1) or ((r_object_id = '$value(r_object_id)' and training_required = 0) or (r_object_id = '$value(r_object_id)' and seq_lms_is_quiz = 0 and training_required = 1 and (r_object_id not in (select child_id from dm_relation where relation_name = 'Link to Training Document') and r_object_id not in (select parent_id from dm_relation where relation_name = 'Link to Training Document'))) or (r_object_id = '$value(r_object_id)' and training_required = 1 and seq_lms_is_quiz = 1 and seq_lms_material_type='SOP' and (seq_lms_quiz_name in ( select object_name from cd_quality_gmp where r_object_id in (select child_id from dm_relation where parent_id = '$value(r_object_id)' and relation_name = 'Link to Training Document')) or seq_lms_quiz_name in (select object_name from cd_quality_gmp where r_object_id in (select parent_id from dm_relation where child_id = '$value(r_object_id)' and relation_name = 'Link to Training Document'))))))

    If the Quiz Name in the Training Info tab and the Object Name of the document it is Related TO is not same, then below message will be displayed on D2:

    The Quiz Name in the Training Info Tab must match the quiz name in the document relationship. Please ensure that the quiz is correctly related to this document using Ã¢â¬ÅLink to Training DocumentÃ¢â¬ï¿½ relationship.
